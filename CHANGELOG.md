## [0.1.0] — 2026-07-08

First packaged release. The project graduated from two standalone scripts (a
proportional reducer and a verification pass, extracted from a real client
engagement) into an installable package with a full CLI.

### Added

- `recpipe` package (`src/recpipe/`) with a stdlib-only core:
  - `records` — byte-safe record splitting on a start-of-line marker;
  - `audit` — read-only survey: per-file counts/sizes/hashes, duplicate-file,
    duplicate-folder and empty-file detection;
  - `reduce` — proportional, distribution-preserving down-sampling (half-up
    rounding, evenly-spaced picks, floor of one record per non-empty file) with
    read-back verification;
  - `transform` — ordered regex substitution rules from a JSON file, applied
    per record, with record-count verification;
  - `validate` — independent re-verification of a finished run (separate code
    path from the writing stages);
  - per-file CSV audit reports for every stage.
- `recpipe` console command with `audit`, `reduce`, `transform`, `validate`
  subcommands; dry-run by default, `--apply` to write, script-friendly exit codes.
- Synthetic sample dataset under `examples/sample_data/` (240 records, generated by
  `examples/generate_sample_data.py`, with a planted duplicate folder and empty file)
  and a demo anonymization rules file.
- Test suite (54 tests) covering core logic and CLI behavior.
- GitHub Actions CI: lint (ruff), tests on Linux + Windows across Python 3.9–3.13,
  and a quickstart smoke job that runs the full pipeline on the sample data.
- Documentation: rewritten README, `docs/ROADMAP.md`, `CONTRIBUTING.md`, issue
  templates; the original case study remains in `docs/CASE_STUDY_EN.md`.
